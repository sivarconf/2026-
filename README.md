## 1. 项目概述

### 1.1 比赛任务详细说明

#### 赛道与场地

- **赛道形状**：圆形赛道，直径约 100cm（胶布铺设）
- **小车尺寸限制**：长 ≤ 25cm，宽 ≤ 15cm
- **禁止事项**：禁止遥控、禁止外挂供电

#### 基础部分（B题）

| 步骤 | 描述 |
|------|------|
| 1 | 小车从起点出发，沿黑线循迹行驶 |
| 2 | 黑线上放置若干张扑克牌，**背面朝上**（灰度传感器检测为大面积白场，视为遮线） |
| 3 | 小车压制扑克牌稳定行驶，**不因扑克牌遮线而失控或停车** |
| 4 | 行驶**一圈**后自动停车 |
| 5 | OLED 显示行驶里程（cm）和用时（秒） |

#### 发挥部分（F题）

| 步骤 | 描述 |
|------|------|
| 1 | 小车从起点出发，沿黑线循迹行驶 |
| 2 | 黑线外侧（圆环外侧地面）放置若干张扑克牌，**正面朝上**（K230 摄像头识别） |
| 3 | 扑克牌数量：1~3张（扩展方案1~5张），花色和点数随机 |
| 4 | 小车行进途中，K230 摄像头实时识别扑克牌，LCD 屏幕显示识别结果（中文花色 + 点数） |
| 5 | 行驶**一圈**后停车，LCD 显示最终识别结果 |

#### 扑克牌放置方式

| 类型 | 放置位置 | 朝向 | 检测方式 | 处理 |
|------|----------|------|----------|------|
| 基础题扑克牌 | 黑线正上方 | 背面朝上 | 灰度传感器（全白场/遮线） | 丢线保护 + 遮线保持角度 |
| 发挥题扑克牌 | 黑线外侧地面 | 正面朝上 | K230 摄像头（AI识别） | K230 算法识别 |

#### 关键设计考量

- **圆形赛道**：路径不是直线，遮线时必须保持原有转向角度沿圆弧行进，否则会偏离轨道
- **K230 识别**：小车行进中摄像头角度不断变化，光照不均，需要多帧投票稳定识别
- **STM32 ↔ K230 串口**：仅负责控制开始（START）和停止（STOP），不承载识别结果数据

### 3.2 软件分层

```
┌─────────────────────────────────────┐
│          user_main.c               │  ← 入口：初始化 + 主循环调度
├─────────────────────────────────────┤
│  app_control.c  │  app_line.c  │   │  ← APP层：状态机/循迹/任务调度
│  app_state.c                            │
├─────────────────────────────────────┤
│ bsp_motor │ bsp_encoder │ bsp_gray │     │  ← BSP层：直接操作硬件

├─────────────────────────────────────┤
│         main.c (CubeMX)           │  ← CubeMX层：HAL初始化/中断
```

**分层原则：**
- `main.c` 只在 USER CODE 区域调用 `User_App_Init()` 和 `User_App_Loop()`
- BSP 层只操作硬件寄存器，不涉及业务逻辑
- APP 层负责状态机、循迹算法、任务调度、参数管理
- 所有硬件宏定义集中在 `user_config.h`

## 4. BSP 驱动层详解

### 4.1 电机驱动 — `bsp_motor.c`

基于 TB6612 双 H 桥驱动芯片。

**引脚：**

| 信号 | 引脚 |
|------|------|
| PWMA（左轮） | PA0 (TIM2_CH1) |
| PWMB（右轮） | PA1 (TIM2_CH2) |
| AIN1/2（左轮方向） | PB12/PB13 |
| BIN1/2（右轮方向） | PB14/PB15 |

**PWM 配置：** 频率 20 kHz（ARR=3599），脉宽 0~3599。

**API：**

| 函数 | 说明 |
|------|------|
| `BSP_Motor_Init()` | 初始化，PWM=0，方向=0 |
| `BSP_Motor_Set(int16_t left, int16_t right)` | 设置左右轮PWM（负值反转） |
| `BSP_Motor_Stop()` | 滑行停止（IN1=0, IN2=0） |
| `BSP_Motor_Brake()` | 刹车（IN1=1, IN2=1） |
| `BSP_Motor_Forward(int16_t pwm)` | 两轮同速前进 |
| `BSP_Motor_Backward(int16_t pwm)` | 两轮同速后退 |

### 4.2 编码器驱动 — `bsp_encoder.c`

使用 TIM3（PA6/PA7）和 TIM4（PB6/PB7）编码器模式，4倍频。

**关键参数（`user_config.h`）：**

| 宏 | 值 | 说明 |
|----|---|------|
| `ONE_LAP_COUNT` | 43000 | 一圈目标编码器累计值 |
| `LEFT_ENCODER_DIR` | -1 | 左轮编码器方向修正 |
| `RIGHT_ENCODER_DIR` | 1 | 右轮方向修正 |
| `ENC_TOTAL_MAX` | ONE_LAP_COUNT×4 | 防止溢出上限 |

**API：**

| 函数 | 说明 |
|------|------|
| `BSP_Encoder_Init()` | 初始化 TIM3/TIM4 编码器模式 |
| `BSP_Encoder_GetLeftDelta()` | 获取左轮增量计数（有方向） |
| `BSP_Encoder_GetRightDelta()` | 获取右轮增量计数（有方向） |
| `BSP_Encoder_Reset()` | 重置编码器计数值 |

### 4.3 灰度传感器驱动 — `bsp_gray.c`

使用软件模拟 I2C（不依赖硬件I2C），避免与 OLED 硬件 I2C 冲突。

**接线：** PA5=SCL，PA4=SDA（均配置为高阻/Analog）
**I2C 地址：** 0x4E（AD1=1, AD0=0）

**API：**

| 函数 | 说明 |
|------|------|
| `BSP_Gray_Init()` | 初始化软件I2C引脚 |
| `BSP_Gray_Ping()` | Ping设备，检测是否在线 |
| `BSP_Gray_GetDigital(uint8_t *val)` | 获取8路数字量（bit0=第1路最左，bit7=第8路最右）|
| `BSP_Gray_GetAnalog(uint8_t ch, uint16_t *val)` | 获取指定通道模拟量（0~4095）|
| `BSP_Gray_Calibrate()` | 触发按键校准 |

**digital 位含义：** bit=0 表示检测到黑场（黑线），bit=1 表示白场。

## 5. APP 应用层详解

### 5.1 系统状态机 — `app_state.c`

```
IDLE ──[上电]──→ READY ──[K3]──→ RUNNING ──[跑完]──→ FINISHED
  ↑                                           │
  └────────────[K4/ERROR]─────────────────────┘
```

**状态定义：**

| 状态 | 含义 |
|------|------|
| `APP_STATE_IDLE` | 初始状态 |
| `APP_STATE_READY` | 就绪，等待启动 |
| `APP_STATE_RUNNING` | 运行中，循迹中 |
| `APP_STATE_ERROR` | 错误（丢线超时） |
| `APP_STATE_FINISHED` | 完成（跑完目标圈数） |

**题目类型：**

| 类型 | 含义 | 行为 |
|------|------|------|
| `QUESTION_B` | 基础题 | 一圈后声光提示，再跑一圈（共两圈） |
| `QUESTION_F` | 挑战题（发挥题） | 一圈后停车，K230识别扑克牌 |

### 5.2 黑线循迹核心 — `app_line.c`

这是整个 STM32 工程最核心的文件，包含循迹算法、丢线保护、编码器一圈计数等。

#### 5.2.1 PID 参数配置

**参数扩大10倍存储**（`int16_t`），OLED显示时除以10，实现±0.1微调。

| 参数组 | BASE_PWM | KP | KI | KD |
|--------|----------|----|----|-----|
| B题（基础题，高速） | 1300 | 11.2 | 0.5 | 6.0 |
| F题（挑战题，低速） | 700 | 6.0 | 0.3 | 4.0 |

> 存储值分别为 1300/112/5/60 和 700/60/3/40。

#### 5.2.2 灰度权重设计

8路灰度传感器，权重（与前进方向相关）：

```
[-40, -28, -20, -15, +15, +20, +28, +40]
  ←  偏左（应左转修正）     偏右（应右转修正） →
```

**误差计算：** 加权平均法 — 找到所有检测到黑线的传感器（digital bit=0），计算权重平均值。

**误差含义：**
- `error < 0`：小车偏左 → 右转修正（减少左PWM，增加右PWM）
- `error > 0`：小车偏右 → 左转修正（增加左PWM，减少右PWM）

#### 5.2.3 PID 控制

```
PWM_out = Kp × error + Ki × integral + Kd × (error - last_error)
```

- **P项**：比例修正
- **I项**：积分消除稳态误差（`app_line.c` 中 `F_KI=3`，B题 `B_KI=5`）
- **D项**：微分抑制震荡

#### 5.2.4 丢线保护策略

| 丢线时长 | 策略 | 说明 |
|----------|------|------|
| < 150ms | 保持上一周期 error，直行 | 短暂丢线 |
| 150ms~300ms（持续白场） | 保持遮线前角度，沿圆弧行进 | **扑克牌遮线专用** |
| 300ms~500ms | 降速 50%~70%，保持方向找线 | 中等丢线 |
| > 600ms | 停车，进入 ERROR 状态 | 长时丢线 |

**扑克牌遮线处理（关键）：** 路径是圆形的，遮线时保持遮线前的偏转角度沿圆弧行进，而不是直行。`last_error` 在遮线期间不更新。

#### 5.2.5 一圈计数

使用编码器累加判定：`s_enc_total = |delta_L| + |delta_R|`

| 题目 | 目标值 | 行为 |
|------|--------|------|
| B题 | ONE_LAP_COUNT×2 | 第一圈完成 → 声光提示1秒 → 第二圈 → 停车 |
| F题 | ONE_LAP_COUNT | 第一圈完成 → 停车，同时发送 STOP 给 K230 |

**待验证：** `ONE_LAP_COUNT=43000` 为初始估算值，**需要实测标定**。让小车跑一圈，记录 OLED 显示的 enc_total 值，填入实际测量结果。

#### 5.2.6 声光提示

| 事件 | 蜂鸣器 | LED |
|------|--------|-----|
| 上电完成 | 80ms 响 | — |
| B题第一圈完成 | 长响 1 秒 | 亮 1 秒 |
| K3 按下（启动） | 30ms 响 | — |

### 5.3 电机控制调度 — `app_control.c`

```c
void App_Control_Task(void)
{
    if (App_State_Get() == APP_STATE_RUNNING)
        App_Line_Task();  // 执行循迹

        BSP_Motor_Stop(); // 非运行状态停止电机
```

### 5.4 用户主入口 — `user_main.c`

**初始化顺序：** Delay → Beep → Key → LED → Motor → Encoder → Gray → OLED → UART → State → Control

**按键处理逻辑（无调参页，PID参数编译期固定，运行时仅切换稳定/竞赛模式）：**

## 6. K230 扑克牌识别详解

### 6.1 硬件与框架

- **硬件：** CanMV K230 v1.6
- **软件框架：** PipeLine（通道0显示，通道2 AI推理）
- **分辨率：** 640×480（显示），224×224（AI输入）
- **分类数：** 55类（BLANK背景 + 54种扑克牌）

### 6.2 分类系统

| 类别 | 数量 | 标签格式 |
|------|------|----------|
| 空白/背景 | 1 | BLANK |
| 黑桃 | 13 | S_A ~ S_K |
| 红桃 | 13 | H_A ~ H_K |
| 梅花 | 13 | C_A ~ C_K |
| 方块 | 13 | D_A ~ D_K |
| 小王 | 1 | JOKER_B |
| 大王 | 1 | JOKER_R |

### 6.3 算法 v16 — 滑动窗口连续帧确认（回退版）

> **2026-05-17 回退说明**：v34 复杂算法（候选池加权积累、花色总和分析、7张难检牌优化等）测试成功率反而不如 v16 简单框架。回退至 v16 连续帧确认，同时保留 v28/v29 的小王相关优化。
>
> 核心目标：在小车行进晃动、角度变化、光照不均的条件下，通过连续帧确认稳定识别扑克牌。

#### 6.3.1 状态机

```
WARMUP ──[预热10帧]──→ TRACKING ──[连续3帧同牌]──→ COOLDOWN ──[冷却20帧]──→ TRACKING
```

- **WARMUP**：开机预热 10 帧，跳过不稳定帧
- **TRACKING**：每帧取 Top1，同一张牌连续出现达到 3 帧即确认
- **COOLDOWN**：确认后冷却 20 帧，防止同一张牌被重复识别。已确认的牌 idx 和 rank 在冷却期内被排除。冷却结束清空排除集合，准备识别下一张牌

#### 6.3.2 核心参数

| 参数 | 值 | 说明 |
|------|----|------|
| MIN_CONFIDENCE | 0.45 | 最低置信度门槛 |
| BLANK_GAP | 0.30 | Top1 必须比 BLANK 高至少此值 |
| TOP_GAP_THRESH | 0.12 | Top1 与 Top2 差距小于此值且 Top1<0.65 时降权（×0.7） |
| TOP1_NOISE_THRESH | 0.55 | Top1 低于此值视为弱识别 |
| CONSECUTIVE_THRESH | 3 | 同一张牌连续出现 N 帧确认 |
| COOLDOWN_FRAMES | 20 | 确认后冷却帧数 |
| WARMUP_FRAMES | 10 | 开机预热帧数 |
| JOKER_GAP_THRESH | 0.25 | S_A 与 JOKER 分差阈值，小于此值交换 softmax |
| JOKER_B_DETECT_MIN | 0.10 | JOKER_B 进入 Boost 的最低分值 |
| JOKER_B_BOOST_MULT | 1.3 | JOKER_B 分值 Boost 系数 |
| JOKER_B_MAX_GAP | 0.25 | Boost 后与 Top1 的最大允许差距 |

#### 6.3.3 算法核心模块

**1. 四关过滤（Recognizer.run）：**
- 第一关：Top1 是 BLANK → 丢弃整帧
- 第二关：Top1 与 BLANK 分差 < BLANK_GAP(0.30) → 丢弃
- 第三关：Top1 置信度 < MIN_CONFIDENCE(0.45) → 丢弃
- 第四关：Top1/Top2 分差 < TOP_GAP_THRESH(0.12) 且 Top1 < 0.65 → Top1 降权 ×0.7

**2. 开机预热（WARMUP_FRAMES=10）：** 前 10 帧识别结果直接跳过，避免相机初始化阶段的不稳定帧。

**3. 连续帧确认（StabilityStateMachine）：**
- TRACKING 状态下，每帧收到 cls_idx 后检查是否与当前候选相同
- 同一张牌连续出现 CONSECUTIVE_THRESH(3) 帧 → 确认该牌，进入 COOLDOWN
- 中途换牌 → 切换候选，计数重置为 1
- 无效帧（cls_idx < 0）→ 冻结当前计数不重置
- 候选被排除时 → 自动从 Top2/3 中寻找下一个有效候选

**4. S_A ↔ JOKER 混淆修正（v28 保留）：**
- 当 Top1=S_A 且 S_A-JOKER 分差 < JOKER_GAP_THRESH(0.25) 时，交换两者 softmax 分值
- 同时检查 JOKER_B 和 JOKER_R，解决大小王被误识别为黑桃A 的问题

**5. JOKER_B Boost 机制（v29/v30 保留）：**
- JOKER_B softmax 分值 >= JOKER_B_DETECT_MIN(0.10) 时，乘以 1.3 放大
- Boost 后与 Top1 差距 >= JOKER_B_MAX_GAP(0.25) 则跳过（防误触发）
- 解决 JOKER_B 自身置信度过低、无法通过识别阈值的问题

**6. 单轮排除机制：**
- 确认一张牌后，该牌 idx 和 rank 加入排除集合
- COOLDOWN 期间（20 帧）排除集合有效，防止同一张牌被重复识别
- COOLDOWN 结束后清空排除集合，所有牌可重新被识别（适用于多张不同牌的连续检测）

#### 6.3.4 识别流程

```
每帧 softmax 输出
    ↓
预热检查（跳过前 10 帧）
    ↓
S_A ↔ JOKER 交换（如果 Top1=S_A 且分差 < 0.25）
    ↓
JOKER_B Boost（如果 >= 0.10 且差距合理）
    ↓
重新排序取 Top1/2/3
    ↓
四关过滤（BLANK / BLANK_GAP / MIN_CONF / TOP_GAP）
    ↓
TRACKING 状态 → 连续帧计数 → 达到 3 帧 → 确认 → 进入 COOLDOWN
COOLDOWN 状态 → 等待 20 帧 → 清空排除集合 → 回到 TRACKING
```

#### 6.3.5 与 v34 的区别

v16 回退版相比 v34 **移除**了以下复杂机制：
- 候选池加权积累（Top2 +0.5 / Top3 +0.3 权重投票）
- 花色总和分析 + 同色花色裁决
- 7 张难检牌专项优化（花色补偿、Top1 保护、低门槛确认）
- COOLDOWN 期间每帧检查候选池

**保留**了以下经过验证的优化：
- S_A ↔ JOKER 混淆修正（v28）
- JOKER_B Boost 机制（v29/v30）
- 中文花色分色显示（黑桃/梅花=黑，红桃/方块=红）

回退理由：v34 引入的多层补偿机制在实际测试中成功率反而不如 v16 的简单连续帧确认。简单的骨架更可靠。

### 6.4 串口协议（仅控制用）

STM32 ↔ K230 串口**仅用于控制 K230 的识别开始和停止**，不承载识别结果数据。

| 方向 | 消息 | 说明 |
|------|------|------|
| STM32 → K230 | `0x01` | START，开始识别记录 |
| STM32 → K230 | `0x02` | STOP，停止识别记录 |
| K230 → STM32 | `0xA1` | ACK START |
| K230 → STM32 | `0xA2` | ACK STOP |

> 识别结果由 K230 直接在 LCD 屏幕上显示，无需回传给 STM32。

## 7. 比赛流程实现

### 7.1 B题（基础题）流程

> **代码逻辑（2026-05-15 修正后）：** B题第一圈完成触发声光各1秒；第二圈完成后直接停车。K230仅在F题启用，B题全程不涉及K230通信。

```
上电 → READY
  │
  ├── K1: 切换稳定模式(S)/竞赛模式(C)
  ├── K2: 切换B题/F题（当前B题）
  │
  └── K3按下 → RUNNING
       │
       ├── 循迹行驶（PID控制）
       ├── 扑克牌遮线：保持遮线前角度沿圆弧行进（丢线保护）
       │
       ├── 第一圈完成（enc_total >= ONE_LAP_COUNT）
       │    ├── 蜂鸣器长响 1 秒
       │    ├── LED 长亮 1 秒
       │    └── 1 秒后自动关闭声光，继续行驶
       │
       └── 第二圈完成（enc_total >= ONE_LAP_COUNT×2）
            ├── 停车
            ├── 蜂鸣器短响 80ms
            ├── 切换 STANDBY 页面
            └── OLED 显示行驶距离（enc_total）和用时
```

**关键行为说明：**
- B题不发送 START/STOP 给 K230，K230保持静默
- 第一圈声光提示用于告知裁判第一圈已完成（计时结束）；第二圈完成后无声光提示，直接停车
- 丢线保护：遇到扑克牌遮线时，150ms 内保持遮线前角度；150ms~300ms 进入圆弧模式；300ms~500ms 降速找线；>600ms 停车报错

### 7.2 F题（发挥题）流程

> **代码逻辑（2026-05-15 修正后）：** F题第一圈完成即停止（仅跑一圈），同时发送 STOP 给 K230；K230停止识别后 LCD 显示最终识别结果。

```
上电 → READY
  │
  ├── K1: 切换稳定模式(S)/竞赛模式(C)
  ├── K2: 切换B题/F题（当前F题）
  │
  └── K3按下 → RUNNING
       │
       ├── BSP_UART_K230_SendStart() → K230开始识别
       ├── 循迹行驶（PID控制）
       ├── 扑克牌遮线：保持遮线前角度沿圆弧行进（丢线保护）
       ├── K230识别扑克牌（面朝上放置在路径上）
       │
       └── 第一圈完成（enc_total >= ONE_LAP_COUNT）
            ├── BSP_UART_K230_SendStop() → K230停止识别
            ├── LCD显示最终识别结果（面额/花色/JOKER）
            ├── 停车
            ├── 蜂鸣器短响 80ms
            ├── 切换 STANDBY 页面
            └── OLED 显示行驶距离（enc_total）和用时
```

**关键行为说明：**
- F题仅跑一圈，第一圈完成即停止，**不跑第二圈**
- K230 在第一圈行驶期间持续识别面朝上的扑克牌；K230发送 STOP 后停止识别并锁定最终结果
- F题无声光提示（第一圈无提示）；停车时蜂鸣器短响 80ms（上电提示音同款）
- 丢线保护：遇到扑克牌遮线时，150ms 内保持遮线前角度；150ms~300ms 进入圆弧模式；300ms~500ms 降速找线；>600ms 停车报错

### 7.3 关键安全机制

| 机制 | 实现位置 | 说明 |
|------|----------|------|
| 丢线超时停车 | `app_line.c` `s_lost_cnt` | >600ms 自动进入 ERROR 状态 |
| PWM 限幅 | `app_line.c` | 防止超过电机承受范围 |
| 最低速度保护 | `app_line.c` | PWM < 100 时强制置 100，防止死区 |
| 编码器溢出保护 | `app_line.c` | `ENC_TOTAL_MAX` 上限 |
| 电机方向保护 | `bsp_motor.c` | 初始化时 PWM=0，防止上电乱转 |

## 8. 调参指南

PID 参数在 `app_line.c` 编译期固定，共 4 组（B竞赛/B稳定/F竞赛/F稳定）。运行时通过 **K1** 在竞赛模式(C)和稳定模式(S)之间切换，OLED 第一行实时显示当前模式。

### 8.1 四组参数一览

| 参数组 | BASE_PWM | KP | KI | KD | 适用场景 |
|--------|----------|----|----|-----|----------|
| B题 竞赛(C) | 1300 | 11.2 | 0.5 | 6.0 | 基础题高速 |
| B题 稳定(S) | 1000 | 8.5 | 0.5 | 4.5 | 基础题保守 |
| F题 竞赛(C) | 700 | 6.0 | 0.3 | 4.0 | 发挥题 K230 识别 |
| F题 稳定(S) | 630 | 5.7 | 0.0 | 4.0 | 发挥题保守 |

> 参数内部扩大 10 倍存储（int16_t），代码中显示值 = 存储值 / 10.0。

### 8.2 修改参数

直接修改 `app_line.c` 中的宏定义（竞赛参数）和 `s_stable_params[]` 数组（稳定参数），重新编译下载。

**调参顺序建议：**
1. `BASE_PWM=600`，`KP=10`，`KD=0` — 小车能缓慢启动
2. 只调 `KP`，每次 ±0.5~1.0，直到直线轻微摆动
3. 加 `KD`，每次 ±0.5~1.0，直到摆动减小
4. 逐步提高 `BASE_PWM`，不要一开始就追求速度
5. F题比 B题慢约 50%，给 K230 留足识别时间 |
