## 2. 硬件架构与模块分工

### 2.1 模块职责表

| 模块 | 职责 | 开发优先级 |
|------|------|------------|
| **STM32F103C8T6** | 底盘实时控制（电机、编码器、灰度循迹、状态机） | ⭐ 第一优先 |
| **K230** | 图像识别（扑克牌）、结果显示、串口通信 | 🔵 第二优先，先独立跑通，再联调 |
| **TB6612** | 双电机 H 桥驱动 | 随 STM32 bring-up |
| **灰度模块（感为）** | 循迹黑线检测，按键自动校准 | 随 STM32 bring-up |
| **编码电机 ×2** | 速度与里程反馈 | 随 STM32 bring-up |
| **OLED** | STM32 状态显示、调试信息 | 第一版可暂缓，用串口调试替代 |
| **4 按键** | 模式切换、启动/停止、校准触发 | 随 STM32 bring-up |
| **蜂鸣器** | 状态提示（上电/完成/错误） | 随 STM32 bring-up |

### 2.2 核心开发原则

```
第一阶段：STM32 底盘独立跑通（灰度循迹 + 电机 + 编码器）
第二阶段：K230 独立识别跑通（屏幕显示结果）
第三阶段：串口联调，识别结果驱动状态机
```

**不要一开始就把所有模块混在一起调**。

## 3. 电源系统说明

### 3.1 电源路径

```
电池 12V（XT60 输入）
│
├── 飞线（已焊好）──→ TB6612 VM（电机电源）
│
└── AJ38 降压模块 ──→ 5V
│
└── 电源开关 ──→ 主板 5V 总线
├── STM32（经板载 3.3V LDO）
├── TB6612 VCC + STBY
├── OLED
├── 灰度传感器
└── 蜂鸣器
```

### 3.2 ⚠️ 安全关键约束

| 操作 | 状态 | 风险 |
|------|------|------|
| 关闭电源开关 | **仅断开 5V 侧** | TB6612 VM 仍可能有 12V |
| 拔电池 | **整车完全断电** | 无风险 |
| 改线/焊接/拔插电机 | 必须先拔电池 | 短路或触电风险 |

**记住：关开关 ≠ 安全。操作硬件前拔电池。**

### 3.3 上电前万用表检查清单

| 测量点 | 预期值 | 实测 | 
|--------|--------|------|
| 5V 与 GND 电阻 | >1kΩ（不短路） | |
| VM 与 GND 电阻 | >1kΩ（不短路） | |
| 3.3V 与 GND 电阻 | >1kΩ（不短路） | |
| 开开关后 5V | ~5V | |
| STM32 3.3V | ~3.3V | |
| TB6612 VCC | ~5V | |
| TB6612 STBY | ~5V | |
| TB6612 VM | ~电池电压（~12V） | |
| 关开关后 TB6612 VCC | ~0V | |

## 4. 引脚映射总表

### 4.1 电机与编码器

| 功能 | STM32 引脚 | 外设 | 连接对象 |
|------|-----------|------|----------|
| 左电机 PWM | PA0 | TIM2_CH1 | TB6612 PWMA |
| 右电机 PWM | PA1 | TIM2_CH2 | TB6612 PWMB |
| 左电机方向1 | PB12 | GPIO Out | TB6612 AIN1 |
| 左电机方向2 | PB13 | GPIO Out | TB6612 AIN2 |
| 右电机方向1 | PB14 | GPIO Out | TB6612 BIN1 |
| 右电机方向2 | PB15 | GPIO Out | TB6612 BIN2 |
| 左编码器 A | PA6 | TIM3_CH1 | 左电机 E1A |
| 左编码器 B | PA7 | TIM3_CH2 | 左电机 E1B |
| 右编码器 A | PB6 | TIM4_CH1 | 右电机 E2A |
| 右编码器 B | PB7 | TIM4_CH2 | 右电机 E2B |

**电机方向定义（前进）：**

| 电机 | IN1 | IN2 |
|------|-----|-----|
| 左（U10） | 1 | 0 |
| 右（U11） | 1 | 0 |

- 停止（滑行）：IN1=0, IN2=0
- 刹车：IN1=1, IN2=1

### 4.2 按键

| 按键 | 引脚 | 按下电平 |
|------|------|----------|
| K1 | PB1 | 0 |
| K2 | PB0 | 0 |
| K3 | PB10 | 0 |
| K4 | PB11 | 0 |

**⚠️ 注意：K1=PB1，K2=PB0，不要搞反。**

### 4.3 蜂鸣器

| 功能 | 引脚 | 备注 |
|------|------|------|
| BEEP | PB3 | 需关闭 JTAG，仅保留 SWD |

### 4.4 I2C 总线（灰度 + OLED）

| 功能 | 引脚 | 备注 |
|------|------|------|
| I2C1_SCL | PB8 (Remap) | SCL1 |
| I2C1_SDA | PB9 (Remap) | SDA1 |
| PA4 | **高阻输入 / Analog** | 不主动驱动 SDA1 |
| PA5 | **高阻输入 / Analog** | 不主动驱动 SCL1 |

**⚠️ 关键：PA4/PA5 连到了同一个 I2C 网络，必须配置为高阻态，否则会干扰 PB8/PB9 通信。**

### 4.5 串口（调试 + K230 联调）

| 功能 | 引脚 | 备注 |
|------|------|------|
| USART1_TX | PA9 | 接外部 RX（调试器 / K230） |
| USART1_RX | PA10 | 接外部 TX（调试器 / K230） |

注意：确保 K230 串口为 3.3V TTL 电平。不要接入 RS232。

### 5.5 Timers → TIM2（PWM 生成）

| 配置项 | 值 | 说明 |
|--------|-----|------|

| Channel 2 | **PWM Generation CH2** | PA1，右电机 |
| Prescaler (PSC) | **0** | 72 MHz ÷ (0+1) = 72 MHz 定时器时钟 |
| Counter Period (ARR) | **3599** | 72 MHz ÷ 3600 = **20 kHz** |
| Auto-reload preload | **Enable** | |
| PWM Mode (CH1&CH2) | **PWM mode 1** | |
| Pulse (初始值) | **0** | ⚠️ 必须为 0，防止上电乱转 |
| CH Polarity | High | |

> **频率计算验证：** 72,000,000 ÷ (0+1) ÷ (3599+1) = 20,000 Hz ✅

## 8. 分阶段开发流程

### 8.1 第一阶段：最小系统验证（预计 1 个晚上）

**目标：** 确认 STM32 能烧录、引脚能控制、串口能打印。

| 步骤 | 操作 | 验证方法 |
|---:|---|---|
| 1 | 创建 CubeMX 工程，按第 5 节配置 | 生成代码，编译通过 |
| 2 | 烧录空工程 | ST-Link 能连接，下载成功 |
| 3 | 写蜂鸣器 GPIO 测试 | PB3 高低电平变化，蜂鸣器响/停 |
| 4 | 写按键 GPIO 读取 | 按下 K1~K4，串口打印按键值；按下应为 0 |
| 5 | 串口 `printf` 测试 | 上位机收到 `Hello STM32` |

**本阶段边界：** 此阶段不涉及电机、编码器、I2C，避免调试范围过大。

### 8.2 第二阶段：电机空载测试（预计 1 个晚上）

**目标：** 确认两个电机能正反转，PWM 输出正常。

> ⚠️ 测试前必须让车轮架空，不接触桌面。TIM2 PWM 初始 `Pulse=0`，方向引脚初始化默认 Low。

| 步骤 | 操作 | 验证方法 |
|---:|---|---|
| 1 | 配置 TIM2 PWM，初始 `pulse=0` | 编译、烧录成功，上电电机不乱转 |
| 2 | 左电机前进，`PWM=300`，持续 1 秒后停止 | 左轮正转 |
| 3 | 左电机后退，`PWM=-300`，持续 1 秒后停止 | 左轮反转 |
| 4 | 右电机前进，`PWM=300`，持续 1 秒后停止 | 右轮正转 |
| 5 | 右电机后退，`PWM=-300`，持续 1 秒后停止 | 右轮反转 |
| 6 | `stop`，方向输入 `IN1=0, IN2=0` | 两轮惰性停止 |
| 7 | `brake`，方向输入 `IN1=1, IN2=1` | 两轮急停 |

**方向修正原则：** 如果方向反了，只改 `bsp_motor.c` 内的方向逻辑，不改引脚表，不改 CubeMX 映射。

### 8.3 第三阶段：编码器验证（预计 1 个晚上）

**目标：** 确认编码器能读数，方向与电机前进方向一致。

| 步骤 | 操作 | 验证方法 |
|---:|---|---|
| 1 | 初始化 TIM3/TIM4 编码器模式 | 程序运行无异常 |
| 2 | 用手转动左轮 | 串口打印 TIM3 `CNT` 值变化 |
| 3 | 用手转动右轮 | 串口打印 TIM4 `CNT` 值变化 |
| 4 | 电机驱动左轮前进 | 串口打印左轮 `delta` 应为正 |
| 5 | 电机驱动右轮前进 | 串口打印右轮 `delta` 应为正 |

**方向修正原则：** 如果 `delta` 为负，修改 `user_config.h` 中对应的 `ENCODER_DIR` 为 `-1`。

### 8.4 第四阶段：I2C 与灰度验证（预计 0.5~1 个晚上）

**目标：** 确认感为灰度传感器 I2C 通信正常，阈值校准有效。

| 步骤 | 操作 | 验证方法 |
|---:|---|---|
| 1 | I2C 地址扫描，范围 `0x03~0x77` | 串口打印发现的设备地址 |
| 2 | 触发感为灰度模块按键校准 | 观察指示灯，确认校准完成 |
| 3 | 读取灰度值 | 白底与黑线数值有明显差异 |
| 4 | 手动偏移传感器位置 | `error` 正负与实际偏移方向一致 |

**实物说明：** 当前灰度模块为感为灰度，可通过板载按键傻瓜式调阈值；不同高度可重新按键校准，暂不作为机械问题处理。

### 8.5 第五阶段：开环低速循迹（预计 1~2 个晚上）

**目标：** 小车能低速沿黑线稳定走完一圈。

| 步骤 | 操作 | 验证方法 |
|---:|---|---|
| 1 | 使用初始参数：`base_pwm=600, Kp=20, Kd=0` | 小车能缓慢启动，不乱冲 |
| 2 | 直线低速测试 | 车身无明显摆动 |
| 3 | 小弯测试 | 能跟随黑线 |
| 4 | 大弯测试 | 不丢线 |
| 5 | 手动遮挡灰度约 0.2 秒 | 不应立刻失控 |
| 6 | 完整一圈 | 能回到起点 |

**调参顺序：**

1. 先只调 `Kp`，每次 `+10`，直到直线轻微摆动。
2. 再加 `Kd`，每次 `+10`，直到摆动减小或消除。
3. 最后逐步提高 `base_pwm`，不要一开始就追求速度。

### 8.6 第六阶段：K230 独立调试 + 串口联调

**目标：** 先让 K230 独立完成识别与显示，再与 STM32 串口通信。

| 步骤 | 操作 | 预期结果 |
|---:|---|---|
| 1 | K230 独立运行识别程序 | 屏幕显示摄像头画面和识别结果 |
| 2 | STM32 ↔ K230 串口收发测试 | 双向通信正常 |
| 3 | 联调协议解析 | STM32 能解析 K230 发来的识别结果 |
| 4 | 识别结果驱动状态机 | 例如：看到牌 → 记下 → 一圈后显示 |

**当前 K230 备注：** K230 会重启，但单独供电也会出现，因此暂时保留该问题；当前判断更可能与未烧录/未稳定运行代码有关，先不作为电源或接线故障定性。

### 8.7 第七阶段：比赛流程整合

按比赛要求实现完整状态机：

```text
出发 → 第一圈（含识别）→ 声光提示 → 第二圈 → 自动停车 → 显示里程和时间
```

## 9. 比赛任务专项设计

### 9.1 基础部分：循迹 + 避牌 + 两圈停车

- 扑克牌背面朝上放在黑线上。
- 对灰度传感器来说，压过牌时数值会突变，可能表现为大面积白色、杂色或短时间丢线。
- 处理策略：丢线短时间内（小于 150 ms）保持上一周期转向值，让车直行碾过去；中等丢线时降速找线；长时间丢线时停车保护。

```c
// 关键的丢线保护（在循迹任务里实现）
if (line_found) {
    last_error = error;
    lost_time = 0;
    // 正常 PD 控制
} else {
    lost_time++;
    if (lost_time < 30) {        // 30 × 5ms = 150ms，短丢线
        // 保持 last_error 不变
    } else if (lost_time < 100) { // 100 × 5ms = 500ms，中等丢线
        base_pwm *= 0.7;          // 降速找线
    } else {
        // 停车，进入错误状态
    }
}
```

### 9.2 发挥部分：识别外围扑克牌

- K230 在小车行进中识别扑克牌正面。
- 推荐串口协议：`$CARD,<花色>,<x>,<y>,<置信度>\n`。
- STM32 收到后存储结果，第一圈结束时在 OLED 显示。
- 注意：识别任务应在第一圈行进中完成，不是停车后再识别。

### 9.3 声光提示设计

| 事件 | 蜂鸣器 | LED/OLED 显示 |
|---|---|---|
| 上电完成 | 1 短 | OLED 显示 `READY` |
| 第一圈完成 | 2 短 | OLED 显示 `LAP1 DONE` |
| 第二圈完成 | 1 长 | OLED 显示 `FINISH` |
| 错误（丢线 > 500 ms） | 3 急促短 | OLED 显示 `ERROR` |
