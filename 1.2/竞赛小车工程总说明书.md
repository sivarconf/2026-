## 1. 项目概述

### 1.1 比赛任务详细说明

#### 赛道与场地

- **赛道形状**：圆形赛道，直径约 100cm（胶布铺设）
- **小车尺寸限制**：长 ≤ 25cm，宽 ≤ 15cm
- **禁止事项**：禁止遥控、禁止外挂供电

#### 基础部分（B题）

| 步骤 | 描述 |
|------|------|
| 1 | 小车从起点出发，沿黑线循迹行驶 |
| 2 | 黑线上放置若干张扑克牌，**背面朝上**（灰度传感器检测为大面积白场，视为遮线） |
| 3 | 小车压制扑克牌稳定行驶，**不因扑克牌遮线而失控或停车** |
| 4 | 行驶**一圈**后自动停车 |
| 5 | OLED 显示行驶里程（cm）和用时（秒） |

#### 发挥部分（F题）

| 步骤 | 描述 |
|------|------|
| 1 | 小车从起点出发，沿黑线循迹行驶 |
| 2 | 黑线外侧（圆环外侧地面）放置若干张扑克牌，**正面朝上**（K230 摄像头识别） |
| 3 | 扑克牌数量：1~3张（扩展方案1~5张），花色和点数随机 |
| 4 | 小车行进途中，K230 摄像头实时识别扑克牌，LCD 屏幕显示识别结果（中文花色 + 点数） |
| 5 | 行驶**一圈**后停车，LCD 显示最终识别结果 |

#### 扑克牌放置方式

| 类型 | 放置位置 | 朝向 | 检测方式 | 处理 |
|------|----------|------|----------|------|
| 基础题扑克牌 | 黑线正上方 | 背面朝上 | 灰度传感器（全白场/遮线） | 丢线保护 + 遮线保持角度 |
| 发挥题扑克牌 | 黑线外侧地面 | 正面朝上 | K230 摄像头（AI识别） | K230 算法识别 |

#### 关键设计考量

- **圆形赛道**：路径不是直线，遮线时必须保持原有转向角度沿圆弧行进，否则会偏离轨道
- **K230 识别**：小车行进中摄像头角度不断变化，光照不均，需要多帧投票稳定识别
- **STM32 ↔ K230 串口**：仅负责控制开始（START）和停止（STOP），不承载识别结果数据

- APP 层负责状态机、循迹算法、任务调度、参数管理
- 所有硬件宏定义集中在 `user_config.h`

## 4. BSP 驱动层详解

### 4.1 电机驱动 — `bsp_motor.c`

基于 TB6612 双 H 桥驱动芯片。

**引脚：**

| 信号 | 引脚 |
|------|------|
| PWMA（左轮） | PA0 (TIM2_CH1) |
| PWMB（右轮） | PA1 (TIM2_CH2) |
| AIN1/2（左轮方向） | PB12/PB13 |
| BIN1/2（右轮方向） | PB14/PB15 |

**PWM 配置：** 频率 20 kHz（ARR=3599），脉宽 0~3599。

**API：**

| 函数 | 说明 |
|------|------|
| `BSP_Motor_Init()` | 初始化，PWM=0，方向=0 |
| `BSP_Motor_Set(int16_t left, int16_t right)` | 设置左右轮PWM（负值反转） |
| `BSP_Motor_Stop()` | 滑行停止（IN1=0, IN2=0） |
| `BSP_Motor_Brake()` | 刹车（IN1=1, IN2=1） |
| `BSP_Motor_Forward(int16_t pwm)` | 两轮同速前进 |
| `BSP_Motor_Backward(int16_t pwm)` | 两轮同速后退 |

### 4.2 编码器驱动 — `bsp_encoder.c`

使用 TIM3（PA6/PA7）和 TIM4（PB6/PB7）编码器模式，4倍频。

**关键参数（`user_config.h`）：**

| 宏 | 值 | 说明 |
|----|---|------|
| `ONE_LAP_COUNT` | 43000 | 一圈目标编码器累计值 |
| `LEFT_ENCODER_DIR` | -1 | 左轮编码器方向修正 |
| `RIGHT_ENCODER_DIR` | 1 | 右轮方向修正 |
| `ENC_TOTAL_MAX` | ONE_LAP_COUNT×4 | 防止溢出上限 |

**API：**

| 函数 | 说明 |
|------|------|
| `BSP_Encoder_Init()` | 初始化 TIM3/TIM4 编码器模式 |
| `BSP_Encoder_GetLeftDelta()` | 获取左轮增量计数（有方向） |
| `BSP_Encoder_GetRightDelta()` | 获取右轮增量计数（有方向） |
| `BSP_Encoder_Reset()` | 重置编码器计数值 |

### 4.3 灰度传感器驱动 — `bsp_gray.c`

使用软件模拟 I2C（不依赖硬件I2C），避免与 OLED 硬件 I2C 冲突。

**接线：** PA5=SCL，PA4=SDA（均配置为高阻/Analog）
**I2C 地址：** 0x4E（AD1=1, AD0=0）

**API：**

| 函数 | 说明 |
|------|------|
| `BSP_Gray_Init()` | 初始化软件I2C引脚 |
| `BSP_Gray_Ping()` | Ping设备，检测是否在线 |
| `BSP_Gray_GetDigital(uint8_t *val)` | 获取8路数字量（bit0=第1路最左，bit7=第8路最右）|
| `BSP_Gray_GetAnalog(uint8_t ch, uint16_t *val)` | 获取指定通道模拟量（0~4095）|
| `BSP_Gray_Calibrate()` | 触发按键校准 |

**digital 位含义：** bit=0 表示检测到黑场（黑线），bit=1 表示白场。

### 4.4 按键驱动 — `bsp_key.c`

| 按键 | 引脚 | 按下电平 |
|------|------|----------|
| K1 | PB1 | 低 |
| K2 | PB0 | 低 |
| K3 | PB10 | 低 |
| K4 | PB11 | 低 |

**API：**

| 函数 | 说明 |
|------|------|

## 5. APP 应用层详解

### 5.1 系统状态机 — `app_state.c`

```
IDLE ──[上电]──→ READY ──[K2]──→ RUNNING ──[跑完]──→ FINISHED
  ↑                                           │
  └────────────[K4/ERROR]─────────────────────┘
```

**状态定义：**

| 状态 | 含义 |
|------|------|
| `APP_STATE_IDLE` | 初始状态 |
| `APP_STATE_READY` | 就绪，等待启动 |
| `APP_STATE_RUNNING` | 运行中，循迹中 |
| `APP_STATE_ERROR` | 错误（丢线超时） |
| `APP_STATE_FINISHED` | 完成（跑完目标圈数） |

**题目类型：**

| 类型 | 含义 | 行为 |
|------|------|------|
| `QUESTION_B` | 基础题 | 一圈后声光提示，再跑一圈（共两圈） |
| `QUESTION_F` | 挑战题（发挥题） | 一圈后停车，K230识别扑克牌 |

### 5.2 黑线循迹核心 — `app_line.c`

这是整个 STM32 工程最核心的文件，包含循迹算法、丢线保护、编码器一圈计数等。

#### 5.2.1 PID 参数配置

**参数扩大10倍存储**（`int16_t`），OLED显示时除以10，实现±0.1微调。

| 参数组 | BASE_PWM | KP | KI | KD |
|--------|----------|----|----|-----|
| B题（基础题，高速） | 1300 | 11.2 | 0.5 | 6.0 |
| F题（挑战题，低速） | 700 | 6.0 | 0.3 | 4.0 |

> 存储值分别为 1300/112/5/60 和 700/60/3/40。

#### 5.2.2 灰度权重设计

8路灰度传感器，权重（与前进方向相关）：

```
[-40, -28, -20, -15, +15, +20, +28, +40]
  ←  偏左（应左转修正）     偏右（应右转修正） →
```

**误差计算：** 加权平均法 — 找到所有检测到黑线的传感器（digital bit=0），计算权重平均值。

**误差含义：**
- `error < 0`：小车偏左 → 右转修正（减少左PWM，增加右PWM）
- `error > 0`：小车偏右 → 左转修正（增加左PWM，减少右PWM）

#### 5.2.3 PID 控制

```
PWM_out = Kp × error + Ki × integral + Kd × (error - last_error)
```

- **P项**：比例修正
- **I项**：积分消除稳态误差（`app_line.c` 中 `F_KI=3`，B题 `B_KI=5`）
- **D项**：微分抑制震荡

#### 5.2.4 丢线保护策略

| 丢线时长 | 策略 | 说明 |
|----------|------|------|
| < 150ms | 保持上一周期 error，直行 | 短暂丢线 |
| 150ms~150ms（持续白场） | 保持遮线前角度，沿圆弧行进 | **扑克牌遮线专用** |
| 150ms~500ms | 降速 50%~70%，保持方向找线 | 中等丢线 |
| > 600ms | 停车，进入 ERROR 状态 | 长时丢线 |

**扑克牌遮线处理（关键）：** 路径是圆形的，遮线时保持遮线前的偏转角度沿圆弧行进，而不是直行。`last_error` 在遮线期间不更新。

#### 5.2.5 一圈计数

使用编码器累加判定：`s_enc_total = |delta_L| + |delta_R|`

| 题目 | 目标值 | 行为 |
|------|--------|------|
| B题 | ONE_LAP_COUNT×2 | 第一圈完成 → 声光提示1秒 → 第二圈 → 停车 |
| F题 | ONE_LAP_COUNT | 第一圈完成 → 停车，同时发送 STOP 给 K230 |

**待验证：** `ONE_LAP_COUNT=43000` 为初始估算值，**需要实测标定**。让小车跑一圈，记录 OLED 显示的 enc_total 值，填入实际测量结果。

#### 5.2.6 声光提示

| 事件 | 蜂鸣器 | LED |
|------|--------|-----|
| 上电完成 | 80ms 响 | — |
| B题第一圈完成 | 长响 1 秒 | 亮 1 秒 |
| K2 按下（启动） | 30ms 响 | — |

### 5.3 电机控制调度 — `app_control.c`

```c
void App_Control_Task(void)
{
    if (App_State_Get() == APP_STATE_RUNNING)
        App_Line_Task();  // 执行循迹
    else
        BSP_Motor_Stop(); // 非运行状态停止电机
}
```

### 5.4 用户主入口 — `user_main.c`

**初始化顺序：** Delay → Beep → Key → LED → Motor → Encoder → Gray → OLED → UART → State → Control

**按键处理逻辑：**

| 按键 | 待机页 | 调参B页 | 调参F页 |
|------|--------|---------|---------|
| K1 | → 调参B页 | → 调参F页 | → 待机页 |
| K2 | 启动RUNNING | 选择下一个参数 | 选择下一个参数 |
| K3 | 切换题目B/F | 当前参数 +1 | 当前参数 +1 |

## 6. K230 扑克牌识别详解

### 6.1 硬件与框架

- **硬件：** CanMV K230 v1.6
- **软件框架：** PipeLine（通道0显示，通道2 AI推理）
- **分辨率：** 640×480（显示），224×224（AI输入）
- **分类数：** 55类（BLANK背景 + 54种扑克牌）

### 6.2 分类系统

| 类别 | 数量 | 标签格式 |
|------|------|----------|
| 空白/背景 | 1 | BLANK |
| 黑桃 | 13 | S_A ~ S_K |
| 红桃 | 13 | H_A ~ H_K |
| 梅花 | 13 | C_A ~ C_K |
| 方块 | 13 | D_A ~ D_K |
| 小王 | 1 | JOKER_B |
| 大王 | 1 | JOKER_R |

### 6.3 算法 v34 — 完整设计

> 核心目标：在小车行进晃动、角度变化、光照不均的条件下，通过多帧投票稳定识别扑克牌。

#### 6.3.1 状态机

```
IDLE ──[收到START]──→ TRACKING ──[确认一张牌]──→ COOLDOWN ──[冷却结束]──→ TRACKING
```

- **IDLE**：等待 START 命令
- **TRACKING**：每帧分析 softmax，进行投票
- **COOLDOWN**：确认后进入冷却期，积累候选池，冷却结束后重新进入 TRACKING

#### 6.3.2 核心参数

| 参数 | 值 | 说明 |
|------|----|------|
| MIN_CONFIDENCE | 0.18 | 最低置信度门槛 |
| BLANK_GAP | 0.10 | Top1 与 BLANK 的最小差距 |
| TOP_GAP_THRESH | 0.08 | Top1 与 Top2 最小差距 |
| TOP1_NOISE_THRESH | 0.50 | Top1 弱识别阈值 |
| VOTE_THRESH | 3 | 确认所需票数 |
| MIN_VOTE_FRAMES | 2 | 确认所需最少帧数 |
| COOLDOWN_FRAMES | 10 | 确认后冷却帧数 |
| JOKER_GAP_THRESH | 0.25 | S_A 与 JOKER 的交换阈值 |
| TOP2_WEIGHT | 0.5 | Top2 投票权重 |
| TOP3_WEIGHT | 0.3 | Top3 投票权重 |
| JOKER_B_BOOST_MULT | 1.3 | JOKER_B 分值放大系数 |
| WARMUP_FRAMES | 5 | 开机预热帧数 |

#### 6.3.3 算法核心模块

**1. 开机预热（WARMUP_FRAMES=5）：** 前5帧识别结果不入候选池，避免初始不稳定帧干扰。

**2. Top1/2/3 提取：** 从 softmax 结果中取前三名（跳过 BLANK），用于后续分析。

**3. S_A ↔ JOKER 混淆修正（v28）：**
- 当 Top1=S_A 且 S_A-JOKER 分差 < 0.25 时，交换两者 softmax 分值
- 解决小王/大王被误识别为黑桃A 的问题

**4. JOKER_B Boost 机制（v29/v30）：**
- JOKER_B softmax 分值 >= 0.10 时，乘以 1.3 放大
- Boost 后与 Top1 差距 >= 0.25 时跳过 Boost（防误触发）
- 解决 JOKER_B 自身置信度过低的问题

**5. 花色总和分析 + 裁决（v33）：**
- 计算每种花色的 softmax 置信度总和
- 当 Top1/2 分差 < 0.20 且同颜色不同花色时（H vs D，S vs C），比较花色总和对进行裁决
- 解决同颜色不同花色分不清的问题

**6. 7张难检牌专项优化（v31）：**

| 难检牌 | 类别索引 |
|--------|----------|
| C_10 | 1 |
| C_6 | 6 |
| C_8 | 8 |
| C_9 | 9 |
| C_K | 12 |
| D_Q | 26 |
| JOKER_B | 40 |

- Top1 保护机制：困难牌 Top1 时允许 Top2 更接近
- 难检牌确认门槛更低（vote_score >= 0.25）
- 花色补偿：梅花/小王的同花色所有牌 softmax 分值累加到目标牌

**7. 候选池加权积累（v25/v26/v27）：**
- 普通牌：Top1 → +1.0票
- 难检牌：Top1 → +1.0票，Top2 → +0.5票，Top3 → +0.3票
- 每帧检查候选池，达到阈值立即确认

**8. 永久排除机制（v16）：**
- `confirmed_cards`：整次运行永久排除已确认的牌
- `cooldown_ranks`：冷却期内临时排除同 rank 的所有花色

#### 6.3.4 投票流程

```
每帧 softmax 输出
    ↓
预热检查（跳过前5帧）
    ↓
计算花色总和分析
    ↓
提取 Top1/2/3
    ↓
S_A ↔ JOKER 交换
    ↓
JOKER_B Boost
    ↓
花色裁决（Top1/2 接近 + 同色冲突时）
    ↓
TRACKING 分支：检查 confirmed_cards / cooldown_ranks，过滤后加权投票
COOLDOWN 分支：积累候选池，检查是否达到阈值
    ↓
达到 VOTE_THRESH → 确认该牌 → 发送结果 → 进入 COOLDOWN
```

#### 6.3.5 识别结果处理

- 每次确认后：在 K230 LCD 上显示中文花色和牌面（如「黑桃A」「红桃7」「小王」）
- 运行结束后：将所有识别结果写入 `/data/recognition_log.txt` 供赛后分析
- 串口仅接收 STM32 的 START/STOP 命令，不回传识别结果数据

### 6.4 串口协议（仅控制用）

STM32 ↔ K230 串口**仅用于控制 K230 的识别开始和停止**，不承载识别结果数据。

| 方向 | 消息 | 说明 |
|------|------|------|
| STM32 → K230 | `0x01` | START，开始识别记录 |
| STM32 → K230 | `0x02` | STOP，停止识别记录 |
| K230 → STM32 | `0xA1` | ACK START |
| K230 → STM32 | `0xA2` | ACK STOP |

> 识别结果由 K230 直接在 LCD 屏幕上显示，无需回传给 STM32。

## 8. 调参指南

### 8.1 B题调参（基础题）

目标：高速稳定走完两圈。

| 参数 | 当前值 | 建议范围 | 调整原则 |
|------|--------|----------|----------|
| BASE_PWM | 1300 | 1000~1500 | 速度上限，越高越快但越容易失控 |
| KP | 11.2 | 8~15 | 先调：直线轻微摆动为止 |
| KI | 0.5 | 0~5 | 消除稳态误差，小车整体偏左/右时调 |
| KD | 6.0 | 3~12 | 后调：消除震荡和过冲 |

**调参顺序：**
1. `BASE_PWM=600`，`KP=10`，`KD=0` — 小车能缓慢启动
2. 只调 `KP`，每次 +5~10，直到直线轻微摆动
3. 加 `KD`，每次 +5~10，直到摆动减小
4. 逐步提高 `BASE_PWM`，不要一开始就追求速度

### 8.2 F题调参（发挥题）

目标：低速稳定，给 K230 足够的识别时间。

| 参数 | 当前值 | 说明 |
|------|--------|------|
| BASE_PWM | 700 | 低于B题，保证稳定 |
| KP | 6.0 | 低于B题 |
| KI | 0.3 | 低于B题 |
| KD | 4.0 | 低于B题 |
